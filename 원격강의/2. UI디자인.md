```
픽셀
    이미지를 구성하는 최소 단위인 네모 모양의 작은 점
뷰
    버튼, 그림, 텍스트 등 화면상 사용자의 눈에 보이는 화면 구성요소
HTML
    웹 페이지를 만드는데 사용하는 기본적인 웹 언어
CSS
    웹 문서의 디자인 요소를 담당하며 글꼴, 색상, 여백, 레이아웃 등을 통해 웹 페이지의 외형을 결정하는 언어
유연한 미디어
    화면에 따라 다양하게 미디어를 맞추어 보여주는 것
유연한 격자
    화면에 따라 레이아웃 영역을 유동적으로 조절하는 것

Android 해상도
    DIP(Device Independent Pixels)
        Android의 기본 독립적 수치
        해상도와 상관없이 같은 크기를 보여줌
    DPI(Dots Per Inch)
        1인치 사각형 안에 들어있는 픽셀의 수
        Android는 160dpi를 기본으로 함
iOS 해상도
    Apple 픽셀 해상도
        디바이스의 화면 크기
    Apple 렌더링 해상도
        일반적인 해상도를 의미
        아이폰은 스마트폰 자체의 픽셀 해상도보다 이미지를 표현할 수 있는 해상도가 훨씬 큼
스마트 콘텐츠 레이아웃
    제약 레이아웃
        제약조건 기반 모델
        정해진 제약 조건을 사용하여 화면을 구성
    리니어 레이아웃
        박스 모델
        한쪽 방향으로 차례대로 뷰를 추가하며 화면을 구성
    상대 레이아웃
        규칙 기반 모델
        상위 개념의 컨테이너나 다른 뷰와의 상대적 위치로 화면을 구성
    프레임 레이아웃
        싱글 모델
        순차적 또는 선택적으로 상위에 있는 내용만 화면에 보여주는 방법
    테이블 레이아웃
        격자 모델
        HTML 웹 개발 시 사용하던 정렬 방식과 유사하나 현재는 많이 사용 X
반응형 웹의 정의와 특징
    각 디바이스에 최적화된 화면을 보여주는 방법이다
    유연한 미디어를 통해 화면에 따른 다양한 미디어를 자동으로 맞춰 보여줌
    유연한 격자를 통해 디바이스 및 해상도에 따라 레이아웃 영역을 조절
    화면 너비 및 높이에 맞는 스타일을 개발할 수 있는 CSS 미디어 쿼리를 통해 작업
```

```
UI 패턴과 목업
    패턴
        정형화된 양식
        모든 정보의 다양한 표현 방식은 UI로 존재하고, UI를 바탕으로 다양한 패턴을 제작함
    패턴의 형태
        UI에 의한 패턴은 사용자가 인지하기 쉽고, 조작이 편리한 일정 패턴을 형성함
        UI 패턴에는 정답이 없으며 애플리케이션의 성격 및 기능이 잘 구현된 것이 좋은 패턴이다
디자인 목업
    최종적으로 만들어질 콘텐츠와 유사한 프로토타입모형
    콘텐츠 개발 시 디자인 작업뿐만 아니라 클라이언트와의 의견 교환에도 활용됨
    무엇을 구현할 것인지 인식할 수 있는 인터랙션의 개념을 포함해야 함
    저작도구
        적절한 목업 제작프로그램을 활용하여 완성도 높은 목업 파일 제작 및 인터랙션 구현
        (프로토파이, 인비전)
```

```
스케일러블 UI 디자인
    Center Inside
        이미지가 축소 및 동일한 크기로 표시되며 원본비율을 유지한 채 이미지뷰의 중앙에 보여줌
    Fix Center
        이미지 원본 비율을 유지한 채 이미지뷰 크기에 맞게 크기를 조절하여 중앙에 보여줌
    Fix End
        이미지를 확대 및 축소하여 이미지뷰 하단에 맞추며 이때 원본 이미지와 동일하지 않다면 상단에 여백이 생김
    Fix Start
        이미지를 확대 및 축소하여 나타낸 후 이미지뷰 상단에 맞추며 이때 가로세로가 동일하지 않기 때문에 하단에 여백이 생김
    레이아웃
        정해진 화면 안에 요소들을 배치하는 스타일
        디자인에 앞서 화면상에 어떻게 배치할 것인지 결정지음
    레이아웃의 배치
        배치될 요소에 대한 전체 리스트를 작성한 후 배치방법을 결정함
        상위 및 하위 세부 개념을 배치할 큰 구획 결정
    유동적 레이아웃
        CSS, 미디어 쿼리 등의 반응형 웹 기술을 통해 디바이스 해상도에 따라 자동으로 최적화됨
        모바일 디바이스 사용 증가로 모바일 퍼스트 트렌드를 나타내는 레이아웃의 대표 개념
    레이아웃의 유형
        수직 수평 구도, 대각선 구도, 원 구도
스케일러블 디자인
    9 Patch
        - 이미지를 정해진 일정 영역만을 잡고 늘림
        - 네 모서리는 그대로 두고 수직 수평개념의 사이즈만 조절하여 전체 이미지를 그대로 유지
    9 Patch의 속성
        - 여백 설정에 따라 9 Patch의 영역이 정해지고 이미지 손상을 최소화 하는 스케일 변화 방식
    Drawable XML
        - 코딩으로 만들어진 여러 가지 그래픽
        - 디자이너 없이도 기본 도형 및 아이콘 디자인을 개발자 혼자 구현 가능함
        - 심플한 디자인이 필요한 부분에서 많이 이용도미
        - 코딩으로 이미지를 수정하고 가벼운 용량과 빠른 작업 속도가 장점
이미지뷰
    이미지 뷰
        - 콘텐츠 구성 시 다양한 이미지가 사용됨에 따라 디바이스 상황에 맞는 작업이 중요시 됨
    이미지뷰 정리
        - Center, Center Crop, Center Inside, Matrix, Fit Center, Fit End, Fit Start, Fit XY
        - 스마트폰 디바이스에 활용할 적절한 이미지 방식을 체크하고 분류함
```

```
에뮬레이터
    다른 프로그램이나 장치를 모방하여 다른 기기상에 완벽하게 구현하는 것을 목표로 하는 컴퓨터의 호환성을 위한 소프트웨어
SVG(Scalable Vector Graphics)
    2차원 벡터 그래픽을 표현하기 위한 XML 기반의 파일 형식
XML(eXtensible Markup Language)
    HTML과 비슷한 문자 기반의 마크업 언어로 데이터를 저장하고 전달하는 목적을 가짐
이미지 네이밍
    - 파일 이름만으로 어떤 이미지 내용인지 직관적으로 알 수 있어야 함
    - IC(아이콘), BTN(버튼), BG(백그라운드), ST(상태) 등 의미를 알 수 있는 약어 사용
픽토그램
    - 이미지만으로 사물, 시설, 행위 등의 개념을 이해할 수 있도록 구체적인 의미를 나타냄
    - 작업 효율성을 높여준다
이미지 형식
    - 일반적으로 JPG, PNG를 많이 사용, 하지만 최근 코딩으로 제작하는 SVG를 많이 사용함
    - SVG파일로 아이콘 및 이미지 구현 시 작업 속도와 시간을 단축할 수 있음
    - Invision 같은 프로토타입 툴에서 활용 및 테스트 가능
```
