# part 1

```
아키텍처 설계 
    - 시스템에 대한 기본 조직 체계
    - 간결성 유지, 추상적인 표현 사용, 시스템에 포함될 것들을 가시화하여 이해당사자 간의 관심사에 따른 모형을 제시할 수 있음 => 원활한 의사소통 수단
아키텍처 구축 프로세스
    - 시스템 환경의 이해 - 요구 사항 분석 - 아키텍처 분석 - 아키텍처 설계 - 검증 및 승인
```

```
비즈니스 아키텍처
    비즈니스 모델 및 구조를 정의하며 비즈니스 모델을 위한 기능과 프로세스를 정의함
    경영 활동과 현재 제안된 IT 투자 간의 관련성 파악이 가능하며 기술 변화가 기업에 미치는 영향을 파악할 수 있음
시스템 아키텍처
    시스템이 어떻게 동작하는지 나타내는 구조를 도식화 함
    구성 기술로 시스템 플랫폼, OS, 기타 소프트웨어, Storage 기술, 이중화, 부하 분산 기술이 있음
데이터 아키텍처
    최상위의 단계에서부터 데이터베이스단계까지 데이터에 관한 모든 구조를 통합하여 연계시킨 아키텍처
    전사적인 아키텍처 구축이 목적
```

```
소프트웨어 디자인 패턴
    특정 문맥, 상황에서 공통으로 발생하는 문제에 대해 재사용 가능한 해결책
    유지보수하기 쉬운 객체지향 시스템을 만들 수 있음
    목적에 따라서 소프트웨어 생성 패턴, 구조 패턴, 행위 패턴 3가지로 분류
```

```
스토리보드
    UX 구현에 수반되는 사용자와 목표, 인터페이스 간 상호작용을 시각화하여 개발자, 디자이너와의 의사소통을 돕는 도구
    완성해야 할 서비스와 예상되는 사용자 경험을 미리 보기 위한 방법론
    실제 개발 단계에서 발생할 수 있는 문제점을 미리 발견하고 대처하기 위해 작성한다
```

```
UI 설계 용어
    Wireframe
        UI 중심의 화면 레이아웃을 설계하며, 선과 뼈대를 만드는 것으로 시각적인 프레임워크, 버튼 배치, 페이지 요소 등의 계층 구조 등 페이지 구조와 구조성 측면의 결과를 통합적으로 적용하는 작업이다.
        Wireframe을 작성하면 개발 구성원과 소통에 용이하며, 시각적인 문서작업, 업무의 방향성 그리고 화면이 어떻게 동작하는지에 대하여 이해를 높일 수 있다
        Wireframe 작성 3단계
            견적 : 예산, 작업일정 등의 대략적인 견적과 화면 흐름에 대한 계획
            기능 : 화면들을 상세하게 설께하여 리스트 업 하고 화면과 플로우에 맞게 컨트롤러들을 디자인 함
            유용성과 디자인 초안 : 사용성을 가다듬고 앱의 전체적인 Look을 결정하며 데이터 모델과 컨트롤러 레이아웃을 반복적으로 검토
    Mockup
        실물과 흡사한 정적인 형태의 모형
        서비스 개발 전에 결과물을 그려보는 것
    Prototype
        다양한 인터랙션을 결합하여 실제 서비스처럼 작동하는 동적인 형태의 모형을 작성하는 것
```

```
User Interface
    좋은 사용자 인터페이스는 심리학과 생리학에 기반하여 사용자가 필요로 하는 요소를 쉽게 찾고 사용하며, 그 요소로 명확하게 의도한 결과를 쉽게 얻어 낼 수 있어야 함
User Experience
    보기 좋은 시각물을 만드는 것보다 풍부한 사용자 경험을 제공하는 것이 목표
    양질의 사용자 경험을 제공하기 위해서 사용자를 이해하고 인지적, 감성적, 행위적 특성을 발견하여 사용자의 NEEDS를 도출한 디자인을 만들어 내야 함
    UX 설계 시 사용자가 어떻게 받아들이고, 어떤 상황에서, 무엇을 느끼며, 어떻게 반응하는지에 대한 관점을 파악해야 함
```

```
UI 내비게이션
    이동 경로나 방법 또는 그것을 돕는 구조와 인터페이스 디자인 등을 포괄하는 표현
    사용자가 원하는 콘텐츠를 빠르고 정확하게 찾을 수 있도록 함
    구성요소
        내비게이션 바, 풀다운 메뉴, 로컬 검색엔진, 버튼/아이콘, 사이트맵, 링크 걸린 텍스트
```

```
인터랙션 디자인
    상호작용 단방향 서비스가 아닌 사용자와 서비스, 제품간의 서로 교류가 가능한 양방향 통신이나 서비스를 의미
    사용자의 편의성과 사용자의 호기심을 불러일으키는 디자인 필요
    대표적인 방법
        애니메이션 효과, 마이크로 인터랙션 등
프로토타입
    개발 접근법 중 하나로써 개발 초기에 시스템의 모형을 간단히 만들어 사용자가 직접 사용함
    개발된 프로토타이핑을 이용하여 사용자는 요구사항을 확인하고 보완점을 제안한다. 그리고 수정 및 보안 작업을 반복하여 요구사항을 만족하게 한다.
```

```
아키텍처 분석
    프로젝트 초반에 문제를 찾아내어 좀 더 좋은 품질의 소프트웨어 아키텍처 설계를 가능하게 함
아키텍처 평가 기법
    품질 속성에 미치는 영향을 판단하여 아키텍처 적합성을 판단 및 평가하는 표준 기법
    가시적 평가와 비가시적 평가로 분류
```

```
UI 설계 
    개발 프로세스
        UX/UI 기획 -> UI 설계문서 작성 -> GUI 디자인 -> GUI 가이드라인 -> 코딩 개발
    우리가 제공하려는 서비스에서 상호작용을 설계하여 좋은 사용성과 상호 작용 수단과 방식을 제공할 수 있도록 설계 되어야 함
    각 설계 단계 별 산출물을 이용하여 이해관계자들 간의 커뮤니케이션 수단으로 사용 가능
```

```
프로토타입 확인 및 피드백
    사용자 중심 개발방법으로 최종 사용자의 요구를 극대화할 수 있으며 개발 시간을 단축 시킬 수 있다
    저수준 프로토타이핑
        스케치, 페이퍼 프로토타입, 와이어 프레임
    고수준 프로토타이핑
        프로토타이핑 툴 이용
```

# part 2

```
웹
    World Wide Web
    텍스트나 그림, 소리, 영상 등과 같은 멀티미디어 정보를 하이퍼텍스트 방식으로 연결함
    웹 클라이언트에서 정보를 요청(Request)하면 웹 서버에서는 요청에 따른 정보를 클라이언트에게 응답(Response)하게 됨
HTML 5
    www 핵심 마크업 언어
    비디오, 오디오 등 다양한 부가 기능과 최신 멀티미디어 콘텐츠를 액티브X(플러그인) 없이 브라우저에서 쉽게 사용할 수 있게 함
    3D, Graphics & Effects 다양한 2차원 및 3차원 그래픽 기능 지원
    CSS3 글씨체, 색상, 배경 등 다양한 스타일 및 이펙트 기능 제공

```

```
HTML 5 글자 태그
    <pre></pre> : HTML 상에서 사용자가 입혁한 대로 출력
    <blockquote></<blockquote> : 인용문 사용
    앵커 태그 = href 이용한 페이지 이동 태그
    글자 형태를 표현하는 태그 : <b>, <i>, <small>, <sub>, <ins>, <del>
```

```
기본 목록 생성
    <ul>, <ol>, <li> 태그 이용
정의 목록 생성
    <dl> : 정의 목록 선언
    <dt> : 용어의 제목
    <dd> : 용어에 대한 설명
테이블 태그
    이전에는 전체 화면을 테이블로 구조화 하는데 사용되었지만 최근에는 주로 실제 표를 만들 때 쓰임
    <caption> : 표의 제목을 나타냄
    <td>, <tr>, <td>
```

```
미디어 태그
    <img>는 src, alt 두 가지 필수 속성을 가짐
    오디오 태그 <audio>
        mp3, wav, ogg 3가지 형식의 파일 포맷을 지원하지만 브라우저에 따라 지원되지 않을 수 있음
        <source> 태그를 사용하여 같은 미디어 요소에 대한 여러 미디어 리소스(파일 형식)를 지정하고, 브라우저에서 선택할 수 있는 대체 미디어 파일을 지정할 수 있다.
    비디오 태그 <video>
        MP4, WebM, Ogg 3가지 형식 지원
```

```
입력양식 태그
    input : 사용자 입력을 위한 양식을 만들 때 form 태그와 같이 사용
    form 태그는 Input, textarea, button, select, option, optgroup, fieldset, label을 하나 이상 포함 가능함
    textarea
        여러 줄 텍스트를 입력할 때 사용하며 maxlength 속성으로 글자 수 제한 가능함
    outgroup
        목록의 항목을 그룹화할 수 있음
    fieldset, legend
        요소를 그룹화하는데 사용하고 그룹화한 캡션은 legend 태그를 이용해 정의한다
        legend 태그는 fieldset과 같이 사용하기도 하지만 독립적으로 사용할 수도 있음
```

```
영역 태그
    div와 span 태그는 웹 페이지의 영역을 정의하는 태그로서 사용된다
    시멘틱 구조 태그
        시멘틱 웹 : 의미론적 웹이라는 뜻. 리소스에 대한 정보와 자원 사이의 관계-의미 정보를 컴퓨터가 처리할 수 있는 형태로 표현함
        header / footer
        nav : 페이지 이동을 위한 메뉴들이 모여 있는 곳
        aside : 사이드에 위치하는 공간으로 보조 메뉴나 광고, 위젯 등 위치
        section : 여러 중심 내용을 감싸는 공간
        article : 글자가 많이 들어가는 부분을 의미하며 컨텐츠의 내용이 들어가는 부분
```

```
CSS3 스타일
    width, margin, padding, font-size, border-width 등과 같은 길이값을 사용
        화면크기가 다양하기 때문에 절대단위는 사용하지 않는 것을 권장, 다른 길이 등록 정보와 관련된 길이인 상대 단위를 사용하자
    색상 단위 : HEX 코드, RGB, RGBA, HSL, HSLA
```
