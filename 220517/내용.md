```
웹에서 템플릿엔진(JSP)을 안 쓰는 추세

1. spring initializr 세팅
    gradle / Java 버전(설치된 버전보다 작은 걸 고름)
    group : com.dw
    Artifact : board
    Description : Board project for Spring Boot
    Dependencies : devtools, lombok, spring web, mybatis, mysql
    설정 완료하고 generate
    workspace에 압축풀기
2. 이클립스에 import하기
    file - import - Existing Gradle(혹은 Maven) Project 선택, spring initializr 파일 있는 폴더로 경로 수정 후 finish
3. 이클립스 패키지 설정
    mybatis 설정 파일인 yaml 파일 수정(디폴트 application.properties)
    templates 삭제
    sqlmap 패키지 생성
    sqlmapper.xml 생성
    banner, logback 추가
    mvc 패키지 생성
    class 파일 생성
    application 실행해서 Hello World 띄우면 설정 완료!
4. 스프링부트에서 JSP 세팅
    스프링부트는 디폴트로 타임리프를 사용하기 때문에 JSP 사용을 위한 별도의 세팅이 필요하다

    프로젝트 이름에서 오른쪽 클릭 - source folder - src/main/webapp 생성
    src/main/webapp 에서 오른쪽 클릭 - new - other - general - folder - WEB-INF 이름으로 생성
    WEB-INF 안에 folder 이름을 views로 생성
    views - new - other - web - JSP File - 파일 이름을 index.jsp 로 생성
        (jsp는 html 안에서 java 코딩이 가능한 파일!)
    bulid.gradle 의 dependencies에 jsp 다운로드함
5. resources 폴더 아래의 static 폴더
    static 폴더 아래에 css, images, js 폴더가 위치함
    js 폴더 아래에 index.js 파일 생성하고 코드 작성
    
회사에서도 마찬가지로 프로젝트를 clone하고 이클립스 들어가서 import하고 시작하면 됨

템플릿 엔진은 프로젝트 파일 안에 존재해야 함
    JSP는 board 프로젝트 안에 있다.
```
