# 기타 용어
```
- 라이브러리
    특정 기능을 하는 코드 뭉치
    개발자가 라이브러리를 코드에 넣고 원하는 기능을 하는 함수를 가져와서 사용할 수 있음

- 프레임워크
    라이브러리를 포함하는 개념
    개발자가 만든 코드를 실행하는데 있어서 그 실행방법이 코드를 사용하는 프레임워크에 따라 다름.

- 스프링
    자바 플랫폼을 위한 오픈소스 애플리케이션 프레임워크
    대한민국 공공기관의 웹 서비스 개발 시 쓰이는 전자정부 표준 프레임워크의 기반 기술로 사용중
```

# Spring Boot
```
- 스프링 프레임워크의 설정이 복잡한 부분을 자동화시킴(실행환경이나 의존성등을 신경쓰지 않고 바로 코딩가능)

- 단독실행 가능(단독 어플리케이션만으로 배포가 된다)
    기존 스프링에서는 서버가 새로 구축되면 프로그램을 설치하고, 수정하고 과정이 많지만 스프링부트는 과정이 간단하다.

- 내장 웹서버
    내장 웹서버(톰캣, 리액터 네티)를 갖고 있어서 별도의 작업 없이 빠르게 서버를 실행할 수 있다.

- 의존성 관리가 편리함(?)
    스프링에선 Freemarker를 사용하기 위해 버전과 호환 그리고 의존성을 확인하는 과정이 필요했는데 스프링부트는 starter를 사용해서 의존성 관리가 편리하다.

- 어플리케이션의 상태를 실시간으로 확인할 수 있다.

- 설정 자동화
    AutoConfig로 복잡한 설정없이 서버를 바로 띄울 수 있고, 외부 라이브러리도 의존성을 추가하여 자동설정을 사용할 수 있다.

```

# Node js
```
- 싱글 스레드 기반 매우 빠른 고성능 서버

- 많은 Connection을 동시에 처리하는 상황에서 성능이 좋음

- restart 시간이 매우 짧아서 프로세스가 종료되어도 빠르게 다시 시작할 수 있고, 빠른 배포나 업그레이드 작업이 가능

- 자바와 같은 프로그래밍 언어보다 생산성이 좋다. 같은 서버 코드를 짜는데 걸리는 시간이 적음.

- 배우기 쉽다(?)
    learning curve가 낮다?

- 프론트와 백엔드 통합
    프론트 개발자들도 js로 서버 백엔드를 개발할 수 있다.

- 서버 플랫폼으로 인기가 높음
    개발자 커뮤니티 활성화되어있음
    npm을 통해 웬만한 기능이 다른 개발자에 의해 구현되어 있음

- 단점) 가독성이 자바에 비해 낮음
    유지보수 어려울 수 있다

- 단점) 에러를 확인하기 위해서 언어가 실행되어야 하는데, 에러가 날 경우 프로세스가 종료된다
    잘 죽는다(?)

- 단점) 싱글 스레드 모델이기 때문에 하나의 작업이 오래 걸리면 전체 시스템 성능이 떨어진다.
```
