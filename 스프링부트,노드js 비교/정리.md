# Spring Boot
```
+ 어플리케이션의 상태를 실시간으로 확인할 수 있다.
+ 다른 사람들과 또는 오랜시간 작업하는 상황에서 코드가 길어졌을 때 잘못된 호출을 막을 수 있음
    (자바는 함수의 타입 입력이 강제되기 때문)
    = 정적 언어의 장점
+ 자바(컴파일러)가 인터프리터보다 압도적으로 빠름
    (소스코드를 먼저 읽으면서 문법 에러를 감지하고 최적화를 수행하기도 함)
    = 컴파일되는 언어의 장점
+ 전자정부프레임워크가 자바로 만들어짐
    (직간접적으로 공공기관과 관련된 일을 하는 회사들은 자바를 사용할 수밖에 없음)

```

# Node js
```
+ 자바보다 생산성이 좋다. 같은 서버 코드를 짜는데 걸리는 시간이 적음.
    => 생산성이 좋다
+ 많은 Connection을 동시에 처리하는 상황에서 성능이 좋음
    (궁금한 것. 단점에 하나의 작업이 오래 걸리면 전체시스템 성능이 떨어진다고 하는데 어떻게 동시처리 성능이 좋은지??)
+ restart 시간이 매우 짧아서 프로세스가 종료되어도 빠르게 다시 시작할 수 있고, 빠른 배포나 업그레이드 작업이 가능
+ 배우기 쉽다
+ 프론트와 백엔드 통합
    프론트 개발자들도 js로 서버 백엔드를 개발할 수 있다.
+ 서버 플랫폼으로 인기가 높음
    개발자 커뮤니티 활성화되어있음
    npm을 통해 웬만한 기능이 다른 개발자에 의해 구현되어 있음

- 한줄한줄 읽는 인터프리터라서 컴파일러보다 느리다.
- 가독성이 자바에 비해 낮음 (유지보수 어려울 수 있다)
- 에러를 확인하기 위해서 언어가 실행되어야 하는데, 에러가 날 경우 프로세스가 종료된다(잘 죽는다고 표현함)
- 싱글 스레드 모델이기 때문에 하나의 작업이 오래 걸리면 전체 시스템 성능이 떨어진다.

```
