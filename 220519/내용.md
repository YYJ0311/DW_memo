로그인할 때 중요한 정보들이 있기 때문에 GET으로 하면 정보가 유출됨 따라서 POST를 사용한다.
    GET은 @RequestBody 사용 불가(json으로 담은 데이터를 body로 받을 수 없다.)
중요한 데이터를 서버에 전송할 때 POST로 데이터를 Body에 숨긴다.
```
```
@RequestParam
    1개의 HTTP 요청 파라미터를 받기 위해서 사용한다.
    기본적으로 반드시 해당 파라미터가 전송되어야 하고, 전송되지 않으면 400 error를 내보낸다.

@RequestBody
    클라이언트가 전송하는 JSON 형태의 HTTP Body를 JAVA Object로 변환시켜주는 역할
    HTTP Get 메소드는 Body가 존재하지 않기 때문에 사용할 수 없다.
    Spring이 body로 받은 json에 대해서 자동으로 객체를 생성하고 set 해준다.

@ModelAttribute
    클라이언트가 전송하는 Body 내용과 파라미터 값들을 생성자나 Setter를 통해 넣기 위해 사용된다.
    추가 검색 필요
