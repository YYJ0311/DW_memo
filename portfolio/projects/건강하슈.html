건강하슈

건강하슈는 대전시민을 대상으로 사용자 주변의 공공체육시설을 확인하고 이용함으로써 건강해졌으면 좋겠다는 의미에서 만든 프로젝트 입니다.

프로젝트는 6월 20일부터 7월 11일까지 22일간 진행했으며,

크게 두 가지 기능으로 나뉩니다. 첫 번째는 공공체육시설을 보여주는 지도 서비스이고, 두 번째는 지도 데이터와 사용자 데이터를 관리하는 관리자 기능입니다.

팀은 총 3명으로, 모두가 프론트와 백엔드를 아울러서 작업했습니다. 그리고 저는 조장으로써 팀의 역할을 분배하고 서비스 전반적인 기능을 조직적으로 구현하기 위해 노력했습니다.



학원에서 7개월동안 배운 것을 바탕으로 기본적인 CRUD를 구현했고, 구글링을 통해 지도와 같은 새로운 기능들을 구현해 보았습니다.

프로젝트 관리는 Github를 사용했는데, 팀 작업을 위해 branch를 나눠서 작업하는 것은 처음이라 처음보는 commit 충돌 이슈에 당황했지만 팀원간의 의사소통과 인터넷 검색으로 해결하는 능력을 기를 수 있었습니다.

일정 관리는 Notion을 이용했는데 한눈에 프로젝트의 진척도를 볼 수 있었고 업무분담도 수월하게 할 수 있었습니다.

AWS RDS로 원격 DB를 만들고 EC2로 배포했습니다.

geojson을 이용한 폴리곤 그리기, geocoder를 이용한 주소->위도/경도 변환 기능, geolocation을 통한 현재 위치 가져오기, 카카오맵 api 사용법(지도와 마커를 배열에 담아 그리고 지우는 기능 등), 디비버를 이용한 엑셀 데이터 insert, excludeUrl을 이용한 회원 권한별 접속 주소 제한 등의 새로운 기능들을 구현해봤습니다.



프로젝트 초기만 하더라도 우리가 과연 기능들을 다 구현할 수 있을까하는 걱정이 있었지만, 하나씩 완성되어가는 모습을 보며 뿌듯했고 새로운 걸 만들 수 있다는 자신감을 얻었습니다.

팀 프로젝트에서 팀 사이 의사소통을 하는 게 정말 중요하고 그 방법도 적절해야 하며, 업무의 분배가 중요함을 느꼈습니다.

Java와 SpringBoot를 이용한 excel 데이터를 다루는 방법이 익숙치 않아서 여러 시행착오들(dependency 버전 맞추기, 많은 패키지와 클래스 해석, 엑셀 파일 오류 등)이 있었고 결국 해결하지 못해 디비버 기능으로 엑셀 데이터를 가져온 것에 아쉬움이 남습니다. 그래도 코드를 계속 보며 익숙해짐을 조금은 느꼈기 때문에 다음에는 할 수 있지 않을까 생각합니다.
