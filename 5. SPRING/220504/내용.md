```
ORM

    resources 폴더에는 html, css, javascript, image 등 java와 관련없는 파일들이 위치한다.

    우리는 properties 파일대신 yaml로 쓸 것임.
```

```
MVC 패턴
html--(URL)--controller--(vo)--service--(vo)--mapper
                                              mapper --(ORM, Mybatis)(vo)--DB
mapper와 연결해주는 MyBatis 파일에서 쿼리를 잘 짜면 service에서 짜는 비즈니스 로직이 줄어든다.
```
```
controller 패키지 아래의 클래스
    1. @RestController 선언해서 스프링에 알려줌
    2. 클래스 아래에 @Autowired로 private한 service와 연결
    3. @GetMapping을 사용해서 url주소를 입력하고 서비스에서 받아서 리턴할 메소드 간단히 작성(메소드 이름은 call 또는 load로 시작)
service 패키지 아래의 클래스
    1. @Service 선언해서 스프링에 알려줌
    2. 클래스 아래에 @Aurowired로 private한 mapper와 연결
    3. mapper에서 받아서 리턴할 메소드 작성(메소드 이름은 get으로 시작)
mapper 패키지 아래의 인터페이스
    1. @Mapper 선언해서 스프링에 알려줌
    2. 사용할 메소드 작성
resources 패키지 아래 MyBatis 담을 패키지 sqlmap를 생성하고 그 아래 Mybatis(xml) 파일
    1. MyBatis 경로 적어줌
    2. mapper 경로 적어줌
    3. <select id="mapper의 메소드 이름" resultType="리턴타입">
            쿼리구현
        </select>
결과 타입을 결정해줄 vo 패키지 아래의 vo클래스
    1. @Data로 getter, setter 사용하게끔 만듦
    2. 사용할 private 변수 선언(이걸로 출력함)
resources 패키지 아래 MyBatis 설정하는 yaml파일
    1. mapper-locations에 MyBatis파일 경로 입력
        ex) classpath:/sqlmap/sqlmapper_*.xml (sqlmap은 resources 아래, yaml파일과 같은 위치에 있음)
    2. type-aliases-package에 vo 패키지 경로 입력
spring
    1. 드라이버 위치 설정
    com.mysql.cj.jdbc.Driver
    2. url 설정
    jdbc:mysql://localhost:3306/dw(DB 데이터베이스에 맞춰서 변경)?useUnicode=true&charaterEncoding=utf-8&serverTimezone=UTC
    3. username - db 유저네임 입력 모를시 디비버에서 확인
    4. password - db사용 비밀번호 입력 (*****잊어버리면 곤란함 찾기 힘듬)
```
