gitignore
    gitignore 폴더에 있는건 무시하고 push된다
    ex) private폴더를 무시하고 싶으면 gitignore파일에 private/ 추가하면 됨

1. git init
2. 본인 깃허브 들어가서 레파지토리 생성
3. git bash로 코드 입력
    git config --global core.autocrlf true
    git add .
    git branch -M main
    git remote add origin 저장소주소
    git push -u origin main 
    까지 하면 깃허브 저장소에 정상적으로 푸쉬된다.
4. spring-workspace 폴더에서 git bash 열고 "git clone 주소" 입력하면 DW_spring 폴더 생성됨
5. file - import - gradle - spring워크스페이스 경로 설정하고 finish하면 자동으로 lombok 같은 설정이 추가된다.
6. 이클립스에서 DW_spring 아래에 setting.gradle 들어가서 이름 수정
7. 프로젝트 이름 오른쪽 클릭 gradle - refresh
8. 이클립스 우측상단 Open Perspective로 git 저장소를 열 수 있다
(이클립스로 커밋 푸쉬도 가능!)

앞으로 DW_spring 을 이용할 것임

Spring
    ""자바""로 "애플리케이션"을 만들기 위한 ""프레임워크""(툴)
    패키지 3개부터 시작(안드로이드도 같음)
        Controller (url주소 요청받는 곳. 로직구현X)
        Service (비즈니스 로직구현)
        VO (getter, setter : lombok 사용)
    MVC 패턴
        서로 영향없이 쉽게 고칠 수 있는 디자인 패턴
        모델, 뷰, 컨트롤러

    @Controller
    @Service
    @Autowired
        : 스프링이 객체를 생성하고 관리 => IoC (제어의 역전)
        스프링이 제어하는 클래스 = Bean 클래스

ORM

속성파일도 필요함
    ser/main/resources 들어가면 application.properties 파일이 있음
    resources 폴더의 properties(또는 yaml)에 계정정보를 적는다.
    resources 폴더에는 html, css, javascript, image 등 java와 관련없는 파일들이 위치한다.

    우리는 properties 파일대신 yaml로 쓸 것임.
        => application.properties 파일 이름을 application.yaml로 바꿈 (properties가 yaml보다 보기 힘들기 때문)

help - eclipe marketplace에서 yaml 검색 - yaml editor 설치(accept, trust하고 이클립스 재시작) - yaml 파일 이미지 Y로 바뀜
