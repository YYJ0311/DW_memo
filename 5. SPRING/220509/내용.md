```
DML(데이터조작언어)
    SELECT, INSERT, DELETE, UPDATE
        (DB에서 UPDATE는 DELETE하고 INSERT의 묶음임)
DDL
    CREATE, DROP, ALTER

1. DDL과 DML의 핵심 차이점
    DDL은 auto COMMIT
    DML은 COMMIT; 입력해야 함

2. MyBatis의 INSERT, DELETE, UPDATE
    자동으로 commit; 됨

자동으로 commit되는 것들은 로직을 잘 짜야 한다. 실수를 돌이킬 수 없음
실수로 아래 코드가 에러나도 위에서 읽어버린 DML은 이미 실행돼서 반영된 상태임!

3. MyBatis의 INSERT, DELETE, UPDATE의 return은 int
    삽입된, 지워진, 업데이트된 행의 수가 return 된다.

4. INSERT, DELETE, UPDATE (DML, 실무에선 트랜잭션이라고 함)
    트랜잭션 : 데이터가 변하는 것
```

```
HTTP의 GET
    데이터를 header(주소)에 심음
    = 입력한 주소가 파라미터로 들어감
HTTP의 POST
    데이터를 body에 심어서 보냄
    body는 숨겨져있다.
        => 회원가입할 때 개인정보가 주소에 보이지 않음(숨겨짐)
        (header는 주소에 보인다.)
    중요한 정보를 보내거나, 데이터를 많이 보낼 때 post 사용
        (header는 보낼 수 있는 데이터의 양이 많지 않다)
```
